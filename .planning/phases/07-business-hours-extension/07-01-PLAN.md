## Objective

祝日パターンを「全曜日共通の1つの営業時間」に変更する。現在は曜日ごとに祝日パターンを設定できるが、ユーザーの要望により、祝日は曜日に関係なく同一の営業時間を適用する。

## Context

### 現状の実装
- `weekly_schedules`テーブル: `day_of_week` (0-6) + `is_holiday_pattern` (bool) で7曜日×2パターン
- 管理画面: 平日タブと祝日タブ、各タブに7曜日分のフォーム
- スロットAPI: 祝日の場合、該当曜日の祝日パターンを検索

### 変更後の仕様
- 祝日パターンは `day_of_week = 0` (または任意の固定値) の1行のみ保存
- 管理画面: 祝日タブは1つの営業時間フォームのみ表示
- スロットAPI: 祝日の場合、曜日に関係なく `is_holiday_pattern = true` の最初の1行を使用

<task id="4">
### 4. スロットAPI修正（単日版）

**ファイル**: `src/app/api/public/slots/route.ts`

祝日判定後のスケジュール取得ロジックを修正。

変更前:
```typescript
const activeSchedule = schedules?.find((s) => s.is_holiday_pattern === isHoliday)
```

変更後（祝日の場合は曜日を無視）:
```typescript
let activeSchedule: typeof schedules[0] | undefined

if (isHoliday) {
  // 祝日: is_holiday_pattern=true の最初の1行を使用（曜日無視）
  const { data: holidaySchedule } = await supabase
    .from("weekly_schedules")
    .select("...")
    .eq("is_holiday_pattern", true)
    .limit(1)
    .single()
  activeSchedule = holidaySchedule
} else {
  // 平日: 該当曜日のis_holiday_pattern=falseを使用
  activeSchedule = schedules?.find((s) => !s.is_holiday_pattern)
}
```

**検証**: 祝日に予約すると、曜日に関係なく同じ営業時間が適用されること
</task>

## Verification Criteria

1. **管理画面**: 祝日タブに1つの営業時間フォームのみ表示される
2. **DB**: 祝日パターン保存後、`is_holiday_pattern = true` の行が1行のみ
3. **API**: 異なる曜日の祝日（例: 月曜の祝日、木曜の祝日）で同じ営業時間が返される
4. **E2E**: 2026/3/20（金曜・春分の日）に予約 → 祝日パターンの営業時間が表示される
