  <files>.github/workflows/e2e.yml</files>
  <action>
`.github/workflows/e2e.yml` を新規作成する。2-job構成で実装する。

**トリガー（ユーザー決定に厳密に従う）:**
- `pull_request` イベント、`branches: [main]` のみ（develop → main PR時に発火）
- developブランチへのpushでは発火しない（pushトリガーは一切含めない）

**Job 1: wait-for-preview**
- `runs-on: ubuntu-latest`
- `patrickedqvist/wait-for-vercel-preview@v1.3.3` を使用
- `token: ${{ secrets.GITHUB_TOKEN }}`（自動提供。VERCEL_TOKENは不要 — リサーチにより確認済み）
- `max_timeout: 300`（5分待機）
- outputsで `preview-url` をJob 2に渡す

**Job 2: e2e**
- `needs: wait-for-preview` で依存
- `runs-on: ubuntu-latest`
- `timeout-minutes: 30`
- steps:
  1. `actions/checkout@v4`
  2. `actions/setup-node@v4` with `node-version: '20'`、`cache: 'npm'`（node_modulesキャッシュ）
  3. `npm ci`（`npm install`ではなくci）
  4. `npx playwright install chromium --with-deps`（chromiumのみ。全ブラウザインストールは禁止）
  5. `npx playwright test` を以下の環境変数付きで実行:
     - `PLAYWRIGHT_BASE_URL: ${{ needs.wait-for-preview.outputs.preview-url }}`
     - `NEXT_PUBLIC_SUPABASE_URL: ${{ secrets.SUPABASE_DEV_URL }}`
     - `NEXT_PUBLIC_SUPABASE_ANON_KEY: ${{ secrets.SUPABASE_DEV_ANON_KEY }}`
     - `SUPABASE_SERVICE_ROLE_KEY: ${{ secrets.SUPABASE_DEV_SERVICE_ROLE_KEY }}`
     - `JWT_CANCEL_SECRET: ${{ secrets.JWT_CANCEL_SECRET }}`
     - `E2E_MEMBER_EMAIL: ${{ secrets.E2E_MEMBER_EMAIL }}`
     - `E2E_MEMBER_PASSWORD: ${{ secrets.E2E_MEMBER_PASSWORD }}`
     - `E2E_ADMIN_EMAIL: ${{ secrets.E2E_ADMIN_EMAIL }}`
     - `E2E_ADMIN_PASSWORD: ${{ secrets.E2E_ADMIN_PASSWORD }}`
     - `CI: true`
  6. `actions/upload-artifact@v4` で `playwright-report/` を保存
     - `if: ${{ !cancelled() }}`（失敗時・キャンセル時でもアップロード）
     - `name: playwright-report`
     - `retention-days: 30`

**重要な注意点:**
- Slack通知、PRコメント投稿、自動マージは延期事項のため一切含めない
- `actions/checkout@v3` 等の旧バージョンは使わない（v4を使用）
- `dotenv.config({ path: '.env.test' })` はplaywright.config.tsにあるがCIでは.env.testが存在しない。GitHub Actions envブロックで環境変数を渡すことで、dotenvが空振りしても問題ない設計
