    1. **マイグレーション作成** (supabase/migrations/20260222200002_meeting_menus_zoom_account.sql):
       - meeting_menusテーブルにzoom_accountカラム追加
       - `ALTER TABLE meeting_menus ADD COLUMN zoom_account CHAR(1) DEFAULT 'A' CHECK (zoom_account IN ('A', 'B'))`
       - 既存レコードはデフォルト'A'

    2. **Zoom本実装** (src/lib/integrations/zoom.ts):
       - 既存のモック実装を完全に置換
       - **環境変数:**
         - ZOOM_ACCOUNT_A_ID, ZOOM_ACCOUNT_A_CLIENT_ID, ZOOM_ACCOUNT_A_CLIENT_SECRET
         - ZOOM_ACCOUNT_B_ID, ZOOM_ACCOUNT_B_CLIENT_ID, ZOOM_ACCOUNT_B_CLIENT_SECRET
       - **LRUキャッシュ** (lru-cacheパッケージ使用、既存):
         - max: 2 (A, B)
         - ttl: 3500 * 1000 (約1時間 - 100秒バッファ)
       - **getZoomAccessToken(accountType: 'A' | 'B')**:
         - キャッシュヒット時はキャッシュから返却
         - POST https://zoom.us/oauth/token (grant_type: 'account_credentials')
         - Basic認証ヘッダー（client_id:client_secret base64）
       - **createZoomMeeting(accountType, params)**:
         - POST https://api.zoom.us/v2/users/me/meetings
         - type: 2 (Scheduled meeting)
         - timezone: 'Asia/Tokyo'
         - settings: host_video, participant_video true, join_before_host false
         - 返却: { zoom_meeting_id, zoom_join_url, zoom_start_url }
       - **deleteZoomMeeting(meetingId)**:
         - DELETE https://api.zoom.us/v2/meetings/{meetingId}
         - アカウント推測: meetingIdからどちらのアカウントか判定が難しいため、両方試行
         - または: bookingsテーブルにzoom_account保存

    **Interface updates:**
    ```typescript
    interface ZoomMeetingParams {
      topic: string;
      start_time: string;
      duration: number;
      accountType: 'A' | 'B';
    }
    interface ZoomMeetingResult {
      zoom_meeting_id: string;
      zoom_join_url: string;
      zoom_start_url: string;
    }
    ```
