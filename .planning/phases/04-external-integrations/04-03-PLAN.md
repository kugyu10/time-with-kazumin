       - cancelBooking(supabase, bookingId, options)
       - options: { isGuest?: boolean, userEmail?: string, userName?: string }
       - **処理順序:**
         1. 予約データ取得（zoom_meeting_id, google_event_id, menu_id, member_plan_id）
         2. ポイント返還（会員のみ、refund_points RPC）
         3. Zoom会議削除（deleteZoomMeeting）
         4. Googleカレンダーイベント削除（deleteCalendarEvent）
         5. 予約ステータス更新（status: 'canceled'）
         6. キャンセルメール送信（sendBookingCancellationEmail）
       - **エラーハンドリング:**
         - Zoom削除失敗: ログ警告、続行
         - Calendar削除失敗: ログ警告、続行
         - メール送信失敗: ログ警告、続行
         - ステータス更新は必ず実行（外部API失敗に関わらず）
       - **返却:** { success: boolean, pointsRefunded?: number }

    2. **会員キャンセルAPI拡張** (src/app/api/bookings/[id]/route.ts):
       - 既存のDELETEハンドラを拡張
       - cancelBooking()を呼び出し
       - 既存のrefund_points呼び出しはcancelBooking内に移動

    3. **ゲストキャンセルAPI拡張** (src/app/api/guest/cancel/[token]/route.ts):
       - 既存のDELETEハンドラを拡張
       - cancelBooking()を呼び出し（isGuest: true）
       - guest_email, guest_nameを渡す

    **Important:**
    - 外部API（Zoom, Calendar, Email）の失敗は予約キャンセルをブロックしない
    - ポイント返還とステータス更新は必須（失敗時は例外をthrow）
    - retryWithExponentialBackoffをZoom/Calendar削除に適用
