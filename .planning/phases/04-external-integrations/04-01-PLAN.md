    1. **マイグレーション作成** (supabase/migrations/20260222200001_oauth_tokens.sql):
       - pgcrypto拡張を有効化 (CREATE EXTENSION IF NOT EXISTS pgcrypto)
       - oauth_tokensテーブル作成:
         - id SERIAL PRIMARY KEY
         - provider TEXT NOT NULL ('google', 'zoom_a', 'zoom_b')
         - access_token_encrypted BYTEA NOT NULL
         - refresh_token_encrypted BYTEA
         - expiry_date TIMESTAMPTZ
         - created_at/updated_at TIMESTAMPTZ
         - UNIQUE constraint on provider
       - encrypt_token/decrypt_token関数（pgp_sym_encrypt/decrypt使用）
       - upsert_oauth_token RPC関数（providerごとにUPSERT）
       - get_oauth_token RPC関数（復号化して返却）

    2. **トークン暗号化ユーティリティ** (src/lib/integrations/oauth/tokens.ts):
       - ENCRYPTION_KEY環境変数を使用
       - saveEncryptedTokens(): oauth_tokensにUPSERT
       - getDecryptedTokens(): oauth_tokensから復号化して取得
       - service_roleクライアント使用（RLSバイパス）

    3. **Google OAuthクライアント** (src/lib/integrations/oauth/google.ts):
       - googleapis.auth.OAuth2インスタンス作成
       - getAuthUrl(): 認証URL生成（access_type: 'offline', prompt: 'consent', calendarスコープ）
       - getTokensFromCode(): 認証コードからトークン取得、DB保存
       - getOAuthClient(): DBからトークン読み込み、setCredentials、'tokens'イベントでDB更新
       - 遅延初期化パターン（ビルド時エラー回避）

    4. **指数バックオフリトライ** (src/lib/utils/retry.ts):
       - retryWithExponentialBackoff<T>(fn, maxRetries=3, baseDelay=1000)
       - 403/429エラー時にリトライ
       - jitter追加（Math.random() * 1000）

    5. **OAuth認証エンドポイント** (src/app/api/admin/oauth/google/route.ts):
       - GET: getAuthUrl()を返却（管理者のみ）
       - 管理者チェック（profilesテーブルのrole確認）

    6. **OAuthコールバック** (src/app/api/admin/oauth/google/callback/route.ts):
       - GET: codeパラメータからgetTokensFromCode()を呼び出し
       - 成功時は/admin/settingsにリダイレクト（OAuth設定完了表示）
       - エラー時は/admin/settings?error=oauth_failedにリダイレクト

    **Important:**
    - googleapis npmパッケージを使用（npm install googleapis）
    - 遅延初期化で環境変数未設定時のビルドエラー回避
    - ENCRYPTION_KEYは32バイト以上必須

    1. **Google Calendar本実装** (src/lib/integrations/google-calendar.ts):
       - 既存のモック実装を本実装に置換
       - **新規追加:**
         - getAdminBusyTimes(startDate, endDate): FreeBusy API呼び出し
         - getCachedBusyTimes(startDate, endDate): LRUキャッシュ付き（15分TTL）
       - **既存関数を本実装に置換:**
         - addCalendarEvent(): Events.insert呼び出し（retryWithExponentialBackoff使用）
         - deleteCalendarEvent(): Events.delete呼び出し（retryWithExponentialBackoff使用）
       - GOOGLE_CALENDAR_ID環境変数でカレンダーID指定（デフォルト: 'primary'）
       - タイムゾーン: 'Asia/Tokyo'

    2. **空きスロットAPI拡張** (src/app/api/public/slots/route.ts):
       - 既存のスロット計算ロジックにbusy時間を統合
       - getCachedBusyTimes()を呼び出し
       - busy時間と重なるスロットを除外
       - OAuthトークンが未設定の場合はbusy時間チェックをスキップ（ログ警告のみ）

    **Implementation details:**
    - google.calendar({ version: 'v3', auth: oauth2Client })
    - calendar.freebusy.query({ requestBody: { timeMin, timeMax, timeZone: 'Asia/Tokyo', items: [{ id: calendarId }] } })
    - LRUキャッシュ: max 100, ttl 15 * 60 * 1000 (15分)
    - キャッシュキー: `${startDate}-${endDate}`

    **Error handling:**
    - OAuthトークン未設定: ログ警告、busy時間チェックスキップ
    - API呼び出し失敗: retryWithExponentialBackoffでリトライ、3回失敗で空配列返却（予約は可能にする）
  </action>
