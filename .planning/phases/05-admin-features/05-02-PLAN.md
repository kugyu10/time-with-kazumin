    1. Server Actions作成 (src/lib/actions/admin/schedules.ts):
       - 'use server'ディレクティブ
       - 各関数内で認証・認可チェック

         - weekly_schedulesテーブルから取得

       updateSchedules(schedules: ScheduleInput[]):
         - Zodでバリデーション
         - 各曜日: day_of_week, start_time, end_time, is_available
         - service_roleでUPSERT（day_of_week + is_holiday_pattern でユニーク）
         - revalidatePath('/admin/schedules')

    2. 営業時間設定ページ (src/app/(admin)/schedules/page.tsx):
       - タブUI: 「平日パターン」「祝日パターン」（shadcn/ui Tabs使用）
       - 各タブで7行のフォーム（日〜土）
       - Server ComponentでgetSchedules()で初期データ取得
       - Client Componentでフォーム操作

    3. 営業時間フォーム (src/components/admin/forms/schedule-form.tsx):
       - 'use client'
       - 7行のフォーム（曜日ごと）
       - 各行: 曜日ラベル、開始時刻Input、終了時刻Input、有効/無効Checkbox
       - 一括保存ボタン
       - React Hook Formで管理

    UI設計（研究で確認済み）:
    - 重量級スケジューラーライブラリは不要
    - シンプルなフォームで十分
    - 曜日名: ['日', '月', '火', '水', '木', '金', '土']

  <action>
    1. 会員管理Server Actions (src/lib/actions/admin/members.ts):
       - 'use server'ディレクティブ
       - 各関数内で認証・認可チェック

       getMembers():
         - profilesテーブルからrole = 'member'を取得
         - member_plansをJOINして現在のポイント残高も取得

       createMember(data):
         - email, full_name, plan_id
         - Supabase Auth API (supabase.auth.admin.createUser)で登録
         - 初期パスワードはランダム生成、パスワードリセットメール送信
         - profilesとmember_plansにレコード作成
         - revalidatePath('/admin/members')

       deactivateMember(userId):
         - profilesのis_activeをfalseに設定
         - または supabase.auth.admin.deleteUser（物理削除）
         - MVP時点では論理削除（is_active = false）を推奨

    2. カラム定義 (src/app/(admin)/members/columns.tsx):
       - カラム: 名前、メール、プラン、ポイント残高、ステータス、アクション
       - アクションドロップダウン: ポイント調整、退会

    3. 会員一覧ページ (src/app/(admin)/members/page.tsx):
       - DataTableで会員一覧表示
       - 新規登録ボタン（Dialogで表示）

    4. 会員登録フォーム (src/components/admin/forms/member-form.tsx):
       - email, full_name, plan_id (プラン選択)
       - プラン一覧はServer Componentから渡す

    5. ポイント調整Server Actions (src/lib/actions/admin/points.ts):
       - adjustPoints(memberPlanId, amount, notes):
         - amount: 正=付与、負=減算
         - notes: 必須（監査証跡）
         - manual_adjust_points() RPC呼び出し
         - revalidatePath('/admin/members')

    6. ポイント調整フォーム (src/components/admin/forms/point-adjust-form.tsx):
       - 'use client'
       - 対象会員表示（読み取り専用）
       - amount入力（数値、正負可）
       - notes入力（必須）
       - 確認ダイアログ付き

    注意:
    - ポイント調整にはnotes必須（研究で確認：監査証跡不足のPitfall対策）
    - 会員招待リンク機能はv2（MVP優先で管理者直接登録）
