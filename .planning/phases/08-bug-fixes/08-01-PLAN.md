    **具体的な実装手順:**
    1. まず `cancelBooking()` 内の booking SELECT クエリを確認し、`menu_id` が取得されているか確認する。取得されていない場合は SELECT に `menu_id` を追加する
    2. Zoom削除の直前に、menu_idからzoom_accountを取得するヘルパーを追加（またはインライン実装）する：
       ```typescript
       // meeting_menuからzoom_accountを取得（anyキャストは既存パターン踏襲）
       const { data: menuData } = await (supabase as any)
         .from("meeting_menus")
         .select("zoom_account")
         .eq("id", booking.menu_id)
         .single()
       const zoomAccountType = (menuData?.zoom_account as "A" | "B") || "A"
       ```

    - このDBクエリ失敗時でも Zoom削除は実行する（アカウントタイプ不明の場合はデフォルト"A"を使用）
    - 非ブロッキング設計（失敗してもキャンセル続行）は維持すること
    - anyキャストは既存パターンを踏襲する（CLAUDE.md準拠）
