<task type="auto">
  <name>Task 2: ゲスト予約API・UI実装</name>
  <files>

  <action>
    1. 空きスロット取得API (src/app/api/public/slots/route.ts):
       - GET /api/public/slots?date=YYYY-MM-DD
       - createClient() (anon/public用) で weekly_schedules と bookings を取得
       - 30分単位でスロット生成、既存予約と重複するスロットを除外
       - 返り値: { slots: Array<{ date, startTime, endTime, available }> }

    2. ゲスト予約作成API (src/app/api/guest/bookings/route.ts):
       - POST /api/guest/bookings
       - body: { email, name, slotDate, startTime, endTime }
       - x-forwarded-for ヘッダーからIP取得（フォールバック: 127.0.0.1）
       - checkGuestRateLimit() でレート制限チェック
       - validateGuestBooking() でバリデーション
       - supabaseServiceRole でbookings INSERT（guest_email, guest_name, menu_id=1(カジュアル30分)）
       - DB EXCLUDE制約エラー(23P01)は「既に予約済み」として返却
       - 成功時: 201 + { booking_id, guest_token } を返却
       - レート制限超過: 429 + X-RateLimit-* ヘッダー

    3. パブリックレイアウト (src/app/(public)/layout.tsx):
       - 認証不要のシンプルレイアウト
       - ヘッダーにサービス名のみ表示

    4. ゲスト予約ページ (src/app/(public)/guest/booking/page.tsx):
       - Server Componentでweekly_schedules取得
       - Client Componentでスロット選択+フォーム入力+送信
       - 既存SlotPicker.tsxを再利用（durationMinutes=30固定）
       - 予約成功後、/guest/booking/success?token=XXX にリダイレクト

    5. ゲスト入力フォーム (src/components/guest/GuestBookingForm.tsx):
       - 名前入力（必須、2-100文字）
       - メールアドレス入力（必須、RFC5322準拠）
       - 選択中のスロット表示
       - 送信ボタン
       - エラーメッセージ表示
       - 送信中はローディング表示
  </action>
