  <name>Task 1: JWTキャンセルトークンとGoogle Calendar URL生成</name>
  <files>
    src/lib/tokens/cancel-token.ts

    src/app/api/guest/cancel/[token]/route.ts
  </files>
  <action>
    1. JWTキャンセルトークン (src/lib/tokens/cancel-token.ts):
       - npm install jose (既にインストール済みの可能性あり、確認)
       - JWT_CANCEL_SECRET 環境変数を使用（なければデフォルト値＋警告ログ）
       - generateCancelToken(bookingId: number, guestEmail: string): Promise<string>
         - payload: { booking_id, email }
         - alg: HS256, exp: 7日間
       - verifyCancelToken(token: string): Promise<{ booking_id: number; email: string } | null>
         - 署名検証失敗または期限切れはnull返却

    2. Google Calendar URL生成 (src/lib/calendar/url-generator.ts):
       - generateGoogleCalendarUrl(params: { title, startTime: Date, endTime: Date, description?, location? }): string
       - dates パラメータ: YYYYMMDDTHHMMSSZ/YYYYMMDDTHHMMSSZ 形式
       - details, location はオプション
       - URL: https://calendar.google.com/calendar/r/eventedit?text=...&dates=...

    3. ゲストキャンセルAPI (src/app/api/guest/cancel/[token]/route.ts):
       - DELETE /api/guest/cancel/[token]
       - verifyCancelToken() でトークン検証
       - 無効なトークン: 401 Unauthorized
       - supabaseServiceRole で bookings.status を 'canceled' に更新
       - booking_id と email の両方が一致する場合のみ更新
       - 既にキャンセル済み: 400 Bad Request
       - 予約開始時刻を過ぎている: 400 Bad Request
       - 成功: 200 OK
  </action>
  <verify>
    npm run build
    # トークン生成・検証のユニットテストは省略（TDDではない）
    # 統合テストは手動で実施
  </verify>
  <done>
    1. generateCancelToken, verifyCancelToken が実装され、JWTを生成・検証できる
    2. generateGoogleCalendarUrl がGoogle Calendar URLを生成する
    3. DELETE /api/guest/cancel/[token] でキャンセルできる
  </done>
