<task type="auto">
  <name>Task 1: キャンセルAPIと処理ロジック実装</name>
  <files>
    src/app/api/bookings/[id]/route.ts
    src/lib/bookings/cancel.ts
  </files>
  <action>
    **cancel.ts:**
    - `cancelBooking(bookingId, supabase)` 関数
    - 処理順序:
      1. 予約取得（member_plans JOINでuser_id確認）
      2. ステータス確認（confirmed/pendingのみキャンセル可）
      3. ポイント返還（`supabase.rpc('refund_points')`）
      4. Zoom会議削除（モック）
      5. Calendar削除（モック）
      6. 予約ステータス更新（'canceled'）
      7. キャンセルメール送信（モック）
    - エラー時は途中状態をログして例外をthrow
    - 返り値: { success: boolean, refunded_points?: number, error?: string }

    **api/bookings/[id]/route.ts:**
    - GET: 予約詳細取得（RLS適用で本人のみ）
    - DELETE: キャンセル処理
      - 認証チェック
      - 権限チェック（本人または管理者）
      - `cancelBooking`呼び出し
      - 成功: { success: true, refunded_points }
      - 失敗: { error: string, code: string }

    エラーハンドリング:
    - 予約なし: 404
    - 権限なし: 403
    - 既キャンセル済み: 400 (already_canceled)
    - 過去の予約: 400 (past_booking)
  </action>
  <verify>
    - TypeScriptコンパイルエラーなし
    - `DELETE /api/bookings/[id]`が認証チェック付きで動作
    - ポイント返還が実行される（point_transactionsにレコード追加）
  </verify>
  <done>
    キャンセルAPIが実装され、ポイント返還処理が動作する
  </done>
</task>
