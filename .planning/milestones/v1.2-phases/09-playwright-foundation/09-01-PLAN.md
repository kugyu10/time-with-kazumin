  <files>playwright.config.ts, .env.test.example</files>

以下の手順でインストールと設定を行う。

**インストール:**
```bash
npm install --save-dev @playwright/test@1.58.2
npx playwright install chromium
```

**playwright.config.ts を新規作成:**
- `dotenv.config({ path: '.env.test' })` を先頭で呼び出し（Pitfall 2対策）
- `baseURL = process.env.PLAYWRIGHT_BASE_URL ?? 'http://localhost:3000'`
- `workers: 1`（STATE.md確定済み: Supabase dev接続数制限対策）
- `fullyParallel: false`
- `retries: process.env.CI ? 2 : 0`
- `forbidOnly: !!process.env.CI`
- `reporter: 'html'`
- `use: { baseURL, trace: 'on-first-retry' }`
- projects設定（project-dependencies パターン）:
  - `{ name: 'setup', testMatch: /auth\.setup\.ts/ }`
  - `{ name: 'chromium', use: { ...devices['Desktop Chrome'] }, dependencies: ['setup'] }`
- webServer: CIのときはundefined、ローカルは `{ command: 'npm run dev', url: 'http://localhost:3000', reuseExistingServer: true }`
- `globalSetup: './e2e/global-setup.ts'`
- `globalTeardown: './e2e/global-teardown.ts'`
- `testDir: './e2e'`

dotenvパッケージはNode.js 20に同梱されていないため `npm install --save-dev dotenv` も実行する。

**注意:** `require.resolve()` ではなく文字列パス `'./e2e/global-setup.ts'` を使用する（TypeScriptプロジェクト向け）。

**.env.test.example を新規作成:**
```
# Playwright E2Eテスト用環境変数（.env.test にコピーして実際の値を設定）

E2E_MEMBER_PASSWORD=TestPassword123!

E2E_ADMIN_PASSWORD=TestPassword123!
# CI環境のみ（ローカルでは不要）
# PLAYWRIGHT_BASE_URL=https://your-vercel-preview.vercel.app
```

<task type="auto">
  <name>Task 2: package.json スクリプト追加と .gitignore 更新</name>
  <files>package.json, .gitignore</files>
  <action>
**package.json の scripts セクションに以下を追加（既存スクリプトを保持）:**
```json
"test:e2e": "playwright test",
"test:e2e:ui": "playwright test --ui",
"test:e2e:debug": "playwright test --debug",
"test:e2e:report": "playwright show-report"
```

**package.json を書き換える前に必ず Read ツールで現在の内容を読む。**

**.gitignore に以下を追記（ファイル末尾に追加）:**
```
# Playwright
e2e/.auth/
playwright-report/
test-results/
```

**.gitignore も書き換える前に必ず Read ツールで現在の内容を読む。**

また、`e2e/` ディレクトリのスキャフォールドとして `e2e/.auth/.gitkeep` を作成する（空ファイル）。これにより git clone 後もディレクトリが存在し Pitfall 3 を回避する。ただし `.gitignore` では `e2e/.auth/` ディレクトリ全体を除外しているため、`.gitkeep` は `e2e/.auth/.gitkeep` をコミット対象に含めるよう `.gitignore` に例外を追加する:
```
e2e/.auth/
!e2e/.auth/.gitkeep
playwright-report/
test-results/
```
