# v1 Requirements - Time with Kazumin

**Core Value:** 気軽にかずみんに会いに行ける予約体験 — 堅苦しいビジネスミーティングの予約ではなく、「かずみん、時間空いてる？」と友だちに声をかける感覚でセッションを予約できること。

---

## v1 Requirements

### Guest（ゲスト向け）

- [ ] **GUEST-01**: ゲストは空き時間を日付指定で閲覧できる
- [ ] **GUEST-02**: ゲストは会員登録なしでカジュアル30分セッションを予約できる（名前+メール入力）
- [ ] **GUEST-03**: ゲストは自分のカジュアル予約をキャンセルできる
- [ ] **GUEST-04**: ゲストは予約完了後、1クリックでGoogleカレンダーに登録できる

### Member（会員向け）

- [ ] **MEMBER-01**: 会員はGoogle認証またはメール/パスワードでログインできる
- [ ] **MEMBER-02**: 会員はメニュー選択してポイント消費で予約できる
- [ ] **MEMBER-03**: 会員は現在のポイント残高を確認できる
- [ ] **MEMBER-04**: 会員は自分の予約一覧を確認できる
- [ ] **MEMBER-05**: 会員は予約をキャンセルしてポイント返還を受けられる
- [ ] **MEMBER-06**: 会員はセッション前日にリマインダーメールを受け取れる

### Admin（管理者向け）

- [ ] **ADMIN-01**: 管理者は曜日別の営業時間を設定できる（祝日は別パターン）
- [ ] **ADMIN-02**: 管理者はGoogleカレンダーと同期して空き時間を自動反映できる
- [ ] **ADMIN-03**: 管理者は全予約一覧を確認・ステータス変更・キャンセルできる
- [ ] **ADMIN-04**: 管理者は会員を招待・登録・退会させることができる
- [ ] **ADMIN-05**: 管理者は会員のポイントを手動で付与/減算できる
- [ ] **ADMIN-06**: 管理者はメニュー（セッション種別）をCRUD管理できる
- [ ] **ADMIN-07**: 管理者はプラン（サブスクプラン）をCRUD管理できる

### System（システム自動処理）

- [ ] **SYS-01**: システムは毎月1日にプランに応じたポイントを自動付与する
- [ ] **SYS-02**: システムは予約確定時にZoom会議を自動生成する（メニューに応じてアカウントA/B使い分け）
- [ ] **SYS-03**: システムはキャンセル時にZoom会議を削除する
- [ ] **SYS-04**: システムは予約確認メールを送信する（ユーザー+管理者宛）
- [ ] **SYS-05**: システムはセッション終了後30分でサンキューメールを送信する（ON/OFF可）
- [ ] **SYS-06**: システムはキャンセル時にキャンセルメールを送信する
- [ ] **SYS-07**: システムは予約時に管理者カレンダーにイベント追加、キャンセル時に削除する

## v2 Requirements（検証後）

## Out of Scope

- **決済機能（Stripe連携）** — MVP後、member_plansにstripe_subscription_id追加で対応
- **詳細キャンセルポリシー** — MVP後、時間制限チェック追加で対応
- **LINE通知** — MVP後、通知チャネル抽象化で対応
- **コーチ複数人対応** — 大規模変更のため将来検討
- **メールテンプレートUI編集** — MVP初期はコードベース管理で十分
- **Zoomアカウント管理UI** — 2アカウント固定のため環境変数管理で十分
- **ユーザーカレンダーへの自動追加** — OAuth審査回避のためURLスキーム方式
- **リアルタイムカレンダー同期** — 15分キャッシュで十分、API効率重視
- **グループセッション** — 現状1on1運用
- **ウェイティングリスト** — 現規模では不要
