# Project Instructions

## Language
常に日本語で質問・回答してください

## ファイル編集ルール

- `docs/` 以下のファイルは基本的に編集しないこと
- 唯一の例外: `docs/基本設計書.md` は基本設計を変更した場合のみ加筆可能

## Git ブランチ命名規則

フォーマット: `{type}/{機能名(英語ケバブケース)}`

type は以下のいずれか:
- `feature` — 新機能
- `refactor` — リファクタリング
- `hotfix` — 緊急修正
- `chore` — その他（設定変更、依存更新など）

例: `feature/user-authentication`, `hotfix/login-error`

## 開発手法

- **TDD（テスト駆動開発）** で進めること
  - Red → Green → Refactor のサイクルを守る
  - 実装コードを書く前に、まずテストを書く
- **ユニットテストカバレッジ 80% 以上** を目標とする

## テスト

- フレームワーク: **Vitest** + **React Testing Library**
- テスト実行: `npm test`
- ウォッチモード: `npm run test:watch`
- カバレッジ確認: `npm run test:coverage`
- テストファイルの配置: `src/__tests__/` または対象ファイルと同階層に `*.test.ts(x)`
